05 / GUARDRAILS

실행 전에 확인해야 할
네 가지 함정

이번 runbook은 코드보다 운영 경계에서 발생할 수 있는 오해를 더 구체적으로 기록합니다. 특히 dataset과 VPC runtime은 로컬에서 조용히 잘못 검증하기 쉬운 지점입니다.

01 / DATASET

raw와 feature dataset은 다르다

적재 스크립트 기본값은 feast_offline_store일 수 있습니다. raw 적재 시 반드시 data_lake_raw를 명시합니다.

02 / WRITE MODE

WRITE_TRUNCATE는 전체 재적재다

실행 전 source artifact와 target table row를 점검합니다. 업로드 검수 없이 재적재하지 않습니다.

03 / NETWORK

Redis는 로컬에서 조회하지 않는다

private VPC 주소이므로 serving image와 app service account를 사용해 클러스터 안에서 materialize와 online query를 실행합니다.

04 / LOCAL ENV

로컬 venv에는 Feast가 없다

로컬 Feast 설치를 전제로 하지 않습니다. 배포된 serving runtime의 registry, staging, Redis secret을 그대로 사용합니다.
