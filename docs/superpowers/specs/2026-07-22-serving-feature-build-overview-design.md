# 서빙 Feature Build 비개발 팀원용 시각화 설계

## 목적

PR #220이 프로젝트 전체에서 바꾼 점을 비개발 팀원이 코드 없이 이해할 수 있게
설명한다. 독자는 `왜 바꿨는지`, `요청이 이제 어디를 지나가는지`, `무엇이 이미
검증됐고 무엇이 운영 전제인지`를 한 번의 읽기 흐름으로 파악할 수 있어야 한다.

## 범위

- `docs/reports/2026-07-22-serving-feature-build-overview.html`에 외부 의존성이
  없는 단일 HTML을 추가한다.
- `docs/README.md`의 서빙 인덱스에 보고서 링크를 추가한다.
- #220 구현과 기존 서빙 계약만 설명한다. ETL 구현, Redis 운영 검증, GKE 배포를
  완료됐다고 표현하지 않는다.

## 독자와 핵심 메시지

| 독자 | 알아야 할 것 |
| --- | --- |
| 기획·발표 팀원 | 호출자가 피처를 조립하던 부담이 서버로 옮겨졌고, 추천 점수의 근거가 일관된 저장소에서 온다. |
| 운영 협업자 | 준비되지 않은 모델·피처 스토어는 정상 응답처럼 보이지 않고 503으로 드러난다. |
| 개발 참여 예정자 | 이 저장소는 serving application을 담당하고, Airflow와 GCP 인프라는 각 소유 저장소가 맡는다. |

문서의 중심 문장은 다음이다.

> 이제 호출자는 사용자와 영상 ID만 보낸다. 서빙 서버가 Feast에서 필요한 정보를
> 조립해 같은 15개 모델 피처로 CTR을 계산한다.

## 정보 구조

1. **Hero / 요약** — 변경 한 줄, PR #220 범위, 구현·운영 준비 상태 범례.
2. **변경 전후** — 이전에는 호출자가 피처를 만들어 보냈고, 현재는 ID만 전송한다는
   책임 이동을 좌우 카드로 비교한다.
3. **새 요청 흐름** — `user_id + video_ids` → 첫 Feast 일괄 조회 → 카테고리별 두 번째
   Feast 일괄 조회 → 15개 피처 조립 → CTR 응답을 인라인 SVG 화살표로 잇는다.
4. **프로젝트를 관통하는 효과** — 호출 단순화, 학습-서빙 일관성, fail-closed 상태,
   관측성의 네 카드로 정리한다.
5. **소유 경계와 운영 전제** — Autoresearch, Autoresearch-airflow,
   Autoresearch-infra의 책임을 구분하고 실제 Redis/GKE smoke가 남았음을 명시한다.
6. **검증 증거와 다음 단계** — 로컬 테스트·Feast 격리 테스트·컨테이너 HTTP smoke·CI는
   완료로, 원천 ETL/materialize와 실제 GKE/Redis smoke는 선행 조건으로 표기한다.

## 표현과 접근성

- 기존 `docs/archive/reports/pipeline-visualization.html`처럼 자체 CSS와 인라인 SVG만
  사용한다. CDN, 이미지, 빌드 도구, 민감한 운영 값은 포함하지 않는다.
- 스크롤 가능한 반응형 레이아웃을 사용한다. 다크 테마이지만 상태는 색만으로 구분하지
  않고 `구현됨`, `운영 전제` 텍스트 배지를 함께 표시한다.
- 도식은 기술 이름을 보조로 두고, 본문은 "정보를 모음", "점수를 계산"처럼 일상어를
  우선한다. 피처 15개는 세부 열거 대신 모델 입력 계약으로 설명한다.

## 정확성 불변조건

- `/rerank` 응답은 CTR 내림차순 정렬이 아니라 요청 `video_ids` 순서를 보존한다.
- Feast 조회는 요청당 두 번의 배치 조회이며, 호출자의 모델 피처 전달은 허용하지 않는다.
- 모델/온라인 FeatureStore/계약이 준비되지 않으면 `/healthcheck`와 유효 요청은 503이다.
- 실제 Redis/GKE 검증은 #210, #218, materialize 준비 후의 운영 게이트다.

## 검증 방법

1. HTML을 파싱해 유효한 문서 구조와 섹션/상태 문구를 확인한다.
2. 헤드리스 브라우저로 데스크톱과 모바일 너비를 렌더링해 오버플로우·가독성을 확인한다.
3. `git diff --check`로 문서 변경의 공백 오류를 확인한다.
